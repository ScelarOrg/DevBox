nodepod - vite HMR test

Repro for the [vite] connecting... hang reported after the tailwind v4 fix shipped in #57. The dev server now boots and renders, but Vite's HMR client never reaches the dev server: the iframe shows [vite] connecting... in the console and edits to source files don't propagate without a manual iframe reload. This page boots a minimal Vite 8 + React-TS app inside nodepod, intercepts window.WebSocket and Vite's HMR events from inside the iframe, and posts them back via postMessage so we can see exactly where the connection stalls.

The edit App.tsx (test HMR) button rewrites the heading inside /app/src/App.tsx via pod.fs.writeFile. If HMR is working, the new heading appears in the iframe within ~1s and the counter keeps its current value. If HMR is broken, nothing changes until a manual reload (and the counter resets when you do).
