nodepod · service worker setup DX

Three demos: default-on SW, a NodepodSWSetupError, and the serviceWorker: false opt-out. Each card runs on its own, click any button.

1 · Happy path: Nodepod.boot() with default-on SW

No swUrl needed. The SW at /__sw__.js is served by examples/serve.js, or by serveSWNode() from @scelar/nodepod/server (see the sibling server.js). 

(click to run)

2 · Loud errors: bad swUrl → NodepodSWSetupError

Point swUrl at something the server doesn't know about (/does-not-exist.js). The preflight HEAD fails and boot() throws a typed error with a framework-aware hint. 

(click to run)

3 · Opt-out: { serviceWorker: false }

Skip SW entirely (SSR, Node tests, host apps that don't need preview iframes). VFS and spawn() still work, you just can't use nodepod.port(n). 

(click to run)
